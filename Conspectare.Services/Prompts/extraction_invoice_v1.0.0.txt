## Critical Rules

1. **Extract EVERYTHING** — Do not skip any field. If data is visible in the document, extract it.
2. **Copy exactly** — Numbers, dates, names must match the document character-for-character. NEVER estimate, round, or invent values.
3. **Dates** → ISO 8601 (YYYY-MM-DD). Romanian formats: DD.MM.YYYY, DD/MM/YYYY, DD-MM-YYYY.
4. **Currency** → ISO 4217 (RON, EUR, USD). Default RON if not specified.
5. **CUI/CIF** — Include "RO" prefix exactly as shown. Set `vat_registered: true` if RO prefix present.
6. **VAT rates** → percentage values: 21 (standard), 11 (reduced), 5 (reduced), 0 (exempt/zero-rate).
7. **Line items** — Extract EACH line individually. Never merge or summarize lines.
8. **Math verification** — Verify: quantity × unit_price = line_total_without_vat. Sum of line totals = tax_exclusive_amount. tax_exclusive_amount + total_vat = tax_inclusive_amount. Flag mismatches.
9. **Amounts** — All monetary values with 2 decimal places. For line items: unit_price and line_total_without_vat are WITHOUT VAT; line_total is WITH VAT.
10. **Document language** — Detect the language of the document and set `document_language` (ISO 639-1: "ro", "en", "fr", "de", etc.).
11. **Document type** — Set `document_type` to standardized value: "invoice", "receipt", or "unknown". Also set `raw_document_type` to the exact text as written on the document (e.g., "FACTURĂ FISCALĂ", "TAX INVOICE", "BON FISCAL").

## VAT Categories (for vat_category field)

- **S** — Standard taxable (21%, 11%, 5%)
- **AE** — Scutit cu drept de deducere (reverse charge, export, intra-Community)
- **E** — Scutit fără drept de deducere (medical, educational, financial)
- **Z** — Cotă zero
- **O** — Neimpozabil (outside VAT scope)

## Special Romanian Mentions to Detect

- "Taxare inversă" / "Reverse charge" → `is_reverse_charge: true`, `vat_category: "AE"`
- "Autofacturare" / "Self-billing" → `is_self_billing: true`
- "Scutit de TVA conform art. 292" → `tax_note` with the article reference, `vat_category: "E"`
- "Regim special de taxare" → capture in `tax_note`
- "Conform contract nr." → capture in `contract_reference`

## Critical: Extract First, Flag Second

ALWAYS extract ALL available data, regardless of any anomalies. Review flags are supplementary quality signals — they NEVER prevent extraction. Extract everything you can see in the document, then add flags for issues.

## Review Flags

Generate flags ONLY for genuine quality issues:
- `missing_field` — A required field (invoice_number, invoice_date, supplier.name, supplier.tax_id, currency) is completely absent from the document
- `calculation_mismatch` — Math doesn't add up (with tolerance ±0.02 per line, ±1.00 for totals)
- `format_anomaly` — Genuinely unusual format (e.g., negative quantity, negative total, unrecognizable date format)
- `low_confidence` — You extracted a value but are genuinely unsure about accuracy (e.g., blurry text, ambiguous characters)

Do NOT generate flags for:
- Dates in the current year (2026) — these are NOT future dates
- ANY VAT rate that appears on the document — all rates are valid (19%, 9%, 5%, 0%, 21%, 11%, 25%, 27%, etc.). Different countries and time periods have different rates. Never flag a VAT rate as anomalous.
- Foreign tax IDs or VAT numbers — these are valid for international invoices
- Non-standard currency — if the document shows it, extract it as-is
- Multiple VAT rates on the same document — this is normal (e.g., restaurants with food at one rate and alcohol at another)

Include `field_path` to indicate which field the flag relates to (e.g., "line_items[2].vat_rate").
